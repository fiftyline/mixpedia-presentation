실행 계획 및 확산 방안

6-1. 데이터 고도화
정보 추출 프롬프트 강화 및 스키마 확장
현재는 타겟팅 유형(DEMO·INTEREST·RETARGETING 등)과 데모 타겟팅(성별·연령대) 정도만 구조화되어 있으나, 실제 집행 데이터에는 키워드·관심사·지역·리타겟팅 조건 등 훨씬 세밀한 타겟팅 정보가 포함되어 있음.
이를 정확하게 파싱하려면 타겟팅 유형별 세부 추출 규칙을 프롬프트에 추가 보완해야 하며, 맥락 의존적인 값 해석 정확도 향상을 위해 보다 성능이 높은 모델 활용 검토 필요
데이터 정확도 향상 — 사용자 수정 피드백 루프
LLM이 구조화한 데이터는 업종 분류·타겟팅·캠페인명 등에서 오분류가 발생할 수 있음. 데모 시스템에 구현된 데이터 수정 요청 기능을 통해 실사용자가 직접 오류를 제보하고,
이를 누적·분석하여 프롬프트 보완 및 재처리에 반영하는 Human-in-the-loop 피드백 루프를 운영하면 데이터 정확도를 지속적으로 향상시킬 수 있음
미디어믹스 데이터 지속 적재
실질적인 벤치마크 가치를 유지하려면, 일정 기간마다 주기적으로 엑셀 파일을 적재할 수 있는 수집 기간 및 리소스가 필요하며, 데이터가 축적될수록 업종별·매체별 통계의 신뢰도가 높아질 수 있음.
6-2. 시스템 고도화
UI/UX 개선
현재 데모 시스템은 핵심 기능 검증에 초점을 맞춰 개발된 프로토타입 수준으로, 실무 사용 환경에서는 검색 필터 구성·결과 정렬·화면 레이아웃 등 여러 UX 측면에서 보완이 필요함.
콘텐츠 강화 — 효율 지표 대시보드
현재는 미디어 믹스 구성(매체·예산·타겟팅)을 조회하는 것이 주된 기능이나, 실무에서 더욱 높은 가치를 지니는 것은 매체별 예상 단가(CPM·CPC)와 예상 CTR·VTR 등의 예측 성과 지표임. 미디어믹스 내에 기재된 예측 성과 값을 추가 구조화하여 업종·매체·타겟팅 조건별 예상 효율을 시각화하는 효율 벤치마크 대시보드를 구축하면, 제안 단계에서의 예상 효율 추정과 매체 조합 검토에 활용할 수 있음. 특히 네이버·Google·Meta·카카오 등 주요 매체는 공개된 레퍼런스가 비교적 풍부하지만 (D.Assistant), 버티컬·서브 매체의 경우 외부에서 참고 가능한 예상 단가나 예측 효율 정보를 구하기가 어렵기 때문에 자사 미디어믹스 기반의 서브 매체 예측 효율 벤치마크는 제안 경쟁력 측면에서 특히 높은 차별화 가치를 가짐
Media Planning Agent — 미디어믹스 초안 생성 기능
축적된 미디어믹스 구조화 데이터를 단순 검색·조회하는 단계를 넘어, 업종·예산·캠페인 목적·타겟 조건을 입력하면 유사 집행 사례를 기반으로 초기 매체 조합과 예산 배분안을 생성하는 기능으로 확장 가능함. 실무자는 완성된 제안안을 바로 받는 것이 아니라, 검토 가능한 초안을 빠르게 확보한 뒤 매체 특성, 광고주 요구사항, 최신 단가 정보를 반영해 보정할 수 있음. 이를 통해 미디어 플래닝 초기 탐색 시간을 줄이고, 제안 방향을 데이터 기반으로 정렬하는 데 활용할 수 있음
