3-1. LLM 기반 미디어 믹스 데이터 구조화

미디어믹스 보고서 엑셀 파일은 규칙 기반 및 일반적인 접근 방법으로는 구조화하기 매우 어려운, 정형 데이터처럼 보이는 비정형 데이터

  • 포맷 비표준 — 파일마다 컬럼명, 헤더 위치, 시트 구성이 다름
  • 언어 혼존 — 한국어·영어가 섞인 컬럼명과 약어(예: P2539, 여성 20대 ~ 30대 등)
  • 맥락 의존적 해석 — 같은 값이라도 컬럼 위치와 주변 맥락에 따라 의미가 달라짐 (예: Product는 광고·상품? 혹은 캠페인 대상 판매 제품?)
OpenAI LLM 구조화 파이프라인 · Batch API · Structured Output · Tool Use
01
컬럼 역할 분류
파일마다 다른 컬럼 구조를 LLM이 역할별로 분류 — 헤더 위치·컬럼명 비표준 문제 해결
Batch API Structured Output
02
미디어 믹스 상세 추출

Batch API Structured Output
03
타겟팅 · 디바이스 · 기간 추출
행별 세부 타겟팅 조건, 디바이스, 집행 기간 파싱 — 맥락 의존적 값 처리
Batch API Structured Output

3-2. LLM 기반 미디어믹스 데이터 임베딩 및 벡터 스토어 구축

LLM기반의 미디어 믹스 구조화 데이터를 시스템에서 검색/조회해 활용 가능하게 재구성

  • 구조화 데이터가 적재된 Google Cloud Platform 환경의 BigQuery에 Google Vertex AI 다국어 임베딩 원격 모델 객체 생성 (text-multilingual-embedding-002)
  • LLM이 출력한 구조화된 원시 데이터를 그대로 사용하는 것이 아닌, 캠페인 정보 요약 관점의 자연어로 변환
  • 자연어로 변환한 데이터를 임베딩 후 별도 데이터셋으로 적재 → 벡터 스토어

3-3. 바이브코딩 활용 데모 시스템 개발

구조화 데이터셋 및 벡터 스토어 구축 완료 후, 이를 서비스로 연결하는 백엔드 API와 프론트엔드를 개발 — 프론트엔드 전체는 Claude Code 바이브코딩으로 구현하여 AI 기반 소프트웨어 개발 방식을 프로젝트 내에서 직접 실증

Python 기반 백엔드
FastAPI
  • 미디어 믹스 구조화 데이터셋 및 벡터 스토어 기반 검색·조회 API 구성
  • 데이터컨설팅팀 테스트/개발 Marketing Mix Model 모듈 통합
  • 벡터 유사도 검색, 필터 조건 처리, 집계 쿼리 등 핵심 엔드포인트 제공

  • 바이브코딩 방법론의 실제 적용 가능성을 프로젝트 결과물로 직접 검증
